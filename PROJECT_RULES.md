# Projektregeln

- Jede Komponente besitzt einen eigenen Ordner; ihre Styles liegen dort.
- Globale Styles enthalten nur Variablen, Reset, Typografie, Animationen und responsive Grundregeln.
- Keine zentrale Monster-Stylesheet-Datei, Base64-Bilder, eingebetteten HTML-Bilder, `clip-path`-Polygone oder abgeschnittenen Ecken.
- UI-Flächen sind rechteckig. Fehlende Grafiken werden als schlichte, klar bezeichnete Platzhalter dargestellt.
- Keine Datei soll unnötig über 500 Zeilen wachsen. Wiederverwendbare Logik wird ausgelagert.
- Keine Duplikate oder ungenutzten Dateien; alle Pfade müssen vom Projekt-Root aus korrekt auflösbar sein.
- Nach jeder Phase werden Build und Start geprüft.
